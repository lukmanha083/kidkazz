### 1.3 Product Creation Flow

```typescript
/**
 * Flow when barcode not found during Good Receipt
 */
async function handleUnknownBarcode(barcode: string, context: GoodReceiptContext) {
  // 1. Search in Product Service
  const product = await productService.findByBarcode(barcode);

  if (product) {
    // Product exists - add to receipt items
    return addToReceiptItems(product, context);
  }

  // 2. Product not found - show Create Product modal
  const createProductModal = showCreateProductModal({
    barcode,
    suggestedSku: await generateSku(),
    onSubmit: async (productData) => {
      // 3. Create product via Product Service
      const newProduct = await productService.createProduct({
        barcode: productData.barcode,
        name: productData.name,
        sku: productData.sku,
        categoryId: productData.categoryId,
        baseUnit: productData.baseUnit,
        trackExpiration: productData.trackExpiration,
        trackBatch: productData.trackBatch,
        pricing: productData.pricing,
        additionalUoms: productData.additionalUoms,
        createdBy: context.userId,
        createdFrom: 'GOOD_RECEIPT' // Track origin
      });

      // 4. Auto-create inventory location for this warehouse
      await inventoryService.createProductLocation({
        productId: newProduct.id,
        warehouseId: context.warehouseId,
        minimumStock: 0,
        maximumStock: null
      });

      // 5. Add to receipt items
      return addToReceiptItems(newProduct, context);
    }
  });
}
```

**Why PURCHASE GR waits for invoice?**

1. **Cost accuracy** - Actual cost comes from supplier invoice, not PO
2. **Price changes** - Supplier may apply discounts or price adjustments
3. **Additional charges** - Shipping, handling fees on invoice
4. **PPN amount** - VAT confirmed on invoice
5. **Proper inventory valuation** - COGS calculation needs accurate cost

**What happens at each stage:**

| Stage | Physical Goods | Inventory System | Accounting |
|-------|---------------|------------------|------------|
| DRAFT | Not received | No change | No entry |
| CONFIRMED | âœ“ Received & checked | No change (pending) | No entry |
| PENDING_INVOICE | In warehouse | No change (waiting) | No entry |
| RECONCILED | In warehouse | âœ“ Stock increased | âœ“ DR: Persediaan, CR: Hutang Dagang |

**Why Opening Balance = Modal Disetor?**

When a business starts, the initial inventory is treated as owner's investment:
- The owner is contributing assets (inventory) to the business
- This increases both Assets (Persediaan) and Equity (Modal Disetor)
- It's NOT a purchase (no A/P created) because no supplier is involved

```
Assets = Liabilities + Equity
```

### 2.2 FEFO (First Expired, First Out) Auto-Selection

When issuing goods, system auto-selects batches with earliest expiration:

```typescript
/**
 * Auto-select batches using FEFO
 */
async function selectBatchesFEFO(
  productId: string,
  warehouseId: string,
  requestedQty: number
): Promise<BatchSelection[]> {
  // Get available batches ordered by expiration date (earliest first)
  const batches = await inventoryBatchRepository.findAvailable({
    productId,
    warehouseId,
    orderBy: 'expirationDate',
    orderDirection: 'ASC'
  });

  const selections: BatchSelection[] = [];
  let remainingQty = requestedQty;

  for (const batch of batches) {
    if (remainingQty <= 0) break;

    const allocateQty = Math.min(batch.availableQuantity, remainingQty);

    selections.push({
      batchId: batch.id,
      batchNumber: batch.batchNumber,
      expirationDate: batch.expirationDate,
      quantity: allocateQty,
      location: batch.location
    });

    remainingQty -= allocateQty;
  }

  if (remainingQty > 0) {
    throw new InsufficientStockError(
      `Insufficient stock for product ${productId}. Requested: ${requestedQty}, Available: ${requestedQty - remainingQty}`
    );
  }

  return selections;
}
```

### 2.3 Good Issue Types

| Type | Destination | Use Case | Creates COGS? |
|------|-------------|----------|---------------|
| **SALES** | Customer | Sales order fulfillment | Yes |
| **TRANSFER_OUT** | Other Warehouse | Inter-warehouse transfer | No |
| **SUPPLIER_RETURN** | Supplier | Return to vendor (defective) | No (reduces A/P) |
| **ADJUSTMENT_OUT** | Internal | Stock count adjustment (negative) | Yes (adjustment) |
| **WRITE_OFF** | Disposal | Damaged, expired, lost goods | Yes (loss) |
| **SAMPLE** | Marketing | Product samples, promotions | Yes (marketing exp) |

### 3.3 Accounting Service Handlers

```typescript
// Handle Good Receipt â†’ Create inventory journal (except PURCHASE)
async function handleGoodReceiptPosted(event: GoodReceiptPostedEvent) {
  if (event.receiptType === 'PURCHASE') {
    // âš ï¸ PURCHASE type: Do NOT create journal entry here!
    // Journal entry is created ONLY after invoice reconciliation
    // See: SUPPLIER_INVOICE_PROCESSING.md
    //
    // At this point:
    // - Physical goods are received and checked
    // - GR status = PENDING_INVOICE
    // - Inventory NOT updated yet (waiting for confirmed cost from invoice)
    // - No accounting entry yet
    //
    // The invoice reconciliation process will:
    // 1. Match invoice with this GR
    // 2. Confirm actual cost from invoice
    // 3. Handle any variances (quantity/price)
    // 4. Update inventory with confirmed cost
    // 5. Create journal entry based on PAYMENT METHOD:
    //    - Cash/Bank: DR Persediaan, CR Kas/Bank
    //    - Credit:    DR Persediaan, CR Hutang Dagang

    console.log(`GR ${event.grnNumber} confirmed. Awaiting invoice reconciliation.`);
    return; // No journal entry created here for PURCHASE type

  } else if (event.receiptType === 'OPENING_BALANCE') {
    // Opening balance â†’ Owner's capital contribution (Setoran Modal)
    await accountingService.createJournalEntry({
      description: `Setoran Modal - Persediaan Awal - ${event.grnNumber}`,
      reference: event.grnNumber,
      sourceService: 'inventory-service',
      sourceReferenceId: event.grnId,
      entryType: 'System',
      lines: [
        {
          accountCode: '1210',  // Persediaan Barang Dagang
          direction: 'Debit',
          amount: event.totalCost,
          warehouseId: event.warehouseId
        },
        {
          accountCode: '3100',  // Modal Disetor
          direction: 'Credit',
          amount: event.totalCost
        }
      ]
    });
  }
}

// Handle Good Issue â†’ Create COGS journal (for sales)
async function handleGoodIssuePosted(event: GoodIssuePostedEvent) {
  if (event.issueType === 'SALES') {
    await accountingService.createJournalEntry({
      description: `COGS - ${event.giNumber}`,
      reference: event.giNumber,
      sourceService: 'inventory-service',
      sourceReferenceId: event.giId,
      lines: [
        {
          accountCode: '5010',  // HPP / COGS
          direction: 'Debit',
          amount: event.totalCost,
          salesChannel: 'POS'   // or from order
        },
        {
          accountCode: '1210',  // Persediaan
          direction: 'Credit',
          amount: event.totalCost
        }
      ]
    });
  } else if (event.issueType === 'WRITE_OFF') {
    await accountingService.createJournalEntry({
      description: `Inventory Write-off - ${event.giNumber}`,
      reference: event.giNumber,
      sourceService: 'inventory-service',
      sourceReferenceId: event.giId,
      lines: [
        {
          accountCode: '6820',  // Beban Kerugian Persediaan
          direction: 'Debit',
          amount: event.totalCost
        },
        {
          accountCode: '1210',  // Persediaan
          direction: 'Credit',
          amount: event.totalCost
        }
      ]
    });
  }
}
```

## 5. Summary

### Key Points

1. **Good Receipt** = Inventory IN (increases stock)
2. **Good Issue** = Inventory OUT (decreases stock)
3. **Inline Product Creation** - Create new products directly from Good Receipt form
4. **FEFO** - Auto-select batches with earliest expiration for issues
5. **Event-Driven** - GRN/GI events trigger accounting journal entries

|-----------------|------------------|
